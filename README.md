## Contribute

The [Contributor Guide](./contributors/guide/CONTRIBUTING.md) provides detailed instruction on how to get your ideas and bug fixes seen and accepted, including:

1. How to [find something to work on](./contributors/guide/CONTRIBUTING.md#find-something-to-work-on)
2. How to [create a pull request](./contributors/guide/CONTRIBUTING.md#creating-pull-requests)
3. How to [code review](./contributors/guide/CONTRIBUTING.md##code-review)

If you're interested in being a contributor and want to get involved in
developing the HAMi code, please see [contribute](./contributors/guide/CONTRIBUTING.md) for
details on submitting patches and the contribution workflow.

## Membership
We encourage all contributors to become members. Learn more about requirements and responsibilities of membership in our [Community Membership doc](./community-membership.md).

If you have made contributions that meet the requirements of becoming HAMi member, [simply file an issue](https://github.com/Project-HAMi/community/issues/new) to apply.
